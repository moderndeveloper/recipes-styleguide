Utilities

Most of the time, CSS is taught to be abstracted into UI Components immediately upon needing it for the design of a site, this approach tends to be bad as a team grows because if you have multiple people working on a project you might have implemented the CSS, but someone who had to refactor it may not know all the inner workings of the CSS you wrote and then you have unused code in the codebase. so rather than searching for CSS in multiple files and praying to the CSS gods that you don't miss the old code, you should use utilities to build your components until you start to see duplication, generally, if you see a component with the same CSS classes 3 times, it's time to refactor into a proper component (component being Atom, Molecule or Organism) the likelihood of you finding duplicated CSS classes in views is much higher than seeing CSS duplicated in your CSS partials which make this allows you to have your code grow naturally over time and easily see when to clean up
