Utilities

Most of the time, CSS is taught to be abstracted into UI Components immediately upon needing it for the design of a site. This approach tends to be bad as a team grows. The reason for this is that when multiple people are working on a project you might have implemented the CSS, but someone who had to refactor it may not know all the inner workings of the CSS you wrote. This results in having unused code in the codebase. So rather than searching for CSS in multiple files and praying to the CSS gods that you don't miss the old code, you should use utilities to build your components until you start to see duplication. Generally, if you see a component with the same CSS classes three times, it's time to refactor into a proper component (being an Atom, Molecule or Organism). The likelihood of you finding duplicated CSS classes in views is much higher than seeing CSS duplicated in your CSS partials; this allows you to have your code grow naturally over time and easily detectable when you need to clean it up.
